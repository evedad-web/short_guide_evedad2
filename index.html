가이드-2 · PowerShell과 Supabase

개발을 하나도 모르는 비개발자를 위한, 두 단어의 친절한 설명서입니다. (앞서 만든 가이드.html의 후속편이에요.)

1. PowerShell이 뭐야?

한 줄 정의: 윈도우에 내장된 "명령을 글자로 입력하는 창". 배포할 때 명령어를 붙여넣던 그 창입니다.

평소엔 마우스로 아이콘을 클릭해서 컴퓨터를 쓰죠. PowerShell은 그 대신 글자(명령어)를 타이핑해서 컴퓨터에게 일을 시킵니다.

왜 굳이 글자로?

개발에선 "코드를 검사하고 → 완성본을 만들고 → 인터넷에 올려라" 같은 여러 단계의 정확한 작업을 시켜야 합니다. 이런 건 마우스 클릭보다 글자 명령 한 줄이 훨씬 빠르고 정확해요. 통역으로 치면, 손짓으로 더듬더듬 전하기보다 정확한 문장으로 지시하는 것과 같습니다.

"터미널"이랑 같은 거야?

"터미널"은 이런 명령 입력 창을 부르는 일반 명칭이고, PowerShell은 그중 윈도우가 기본으로 주는 제품 이름입니다. (맥에선 'Terminal'·'bash' 같은 비슷한 다른 프로그램을 씀.) 그래서 "터미널에 붙여넣으세요" = 윈도우에선 PowerShell 창.

우리가 PowerShell로 한 일

명령어무슨 뜻
npm run build코드 검사 + 실행용 완성본 만들기
npx vercel --prod완성본을 인터넷에 배포(업로드)
$env:NODE_TLS_REJECT_UNAUTHORIZED="0""이번 창에서만 회사망 보안검사를 무시" — 회사망이 막아서 필요한 임시 설정

위험한 거 아냐?

명령어를 직접 안 쓰고 받은 걸 그대로 붙여넣는 한 안전합니다. 다만 출처 모를 명령어(인터넷에서 주운 것 등)를 함부로 붙여넣는 건 피하세요. 제가 드리는 배포 명령만 쓰시면 됩니다.

2. Supabase가 뭐야?

한 줄 정의: 인터넷에 있는 "데이터 저장 창고(데이터베이스)" 서비스. 예전에 구글 스프레드시트로 관리하던 기기 현황이 이제 여기 들어 있습니다.

데이터베이스가 뭔데?

자료를 체계적으로 저장·관리하는 곳입니다. 똑똑한 엑셀이라고 생각하면 쉬워요. 표(행과 열)에 자료를 담는 건 엑셀과 비슷한데, 규칙·계산·동시 접속을 훨씬 안정적으로 처리합니다.

왜 엑셀/구글시트 대신?

  • 여러 명이 동시에 대여·반납해도 안 꼬임
  • 화면의 "사용 가능 30, 사용 불가 1" 숫자를 실시간 자동 계산
  • 앱(Vercel)과 모든 팀원이 똑같은 최신 데이터를 봄 (인터넷 클라우드에 있으니까)

우리 Supabase 안에 든 것

"테이블"(엑셀의 시트 같은 것) 형태로 저장돼 있습니다.

테이블담긴 내용
기기 목록번호 · 위치 · 비고 · 사용 불가 여부
대여 기록누가 · 언제 · 무엇을 빌리고 반납했는지
액세서리 보유량이어폰 · 핀마이크 · 스틱마이크 수량

그리고 "뷰(view)"라는 게 있어, "재고 = 총 보유량 − 현재 사용 중"처럼 자동 계산된 값을 화면에 바로 보여줍니다. 그래서 따로 숫자를 세지 않아도 항상 정확해요.

RLS(보안 규칙)

누가 데이터를 읽고 쓸 수 있는지 정하는 잠금장치입니다. 지금은 사내 내부용이라 잠금을 풀어둔 상태예요. 나중에 로그인을 붙이면 여기를 조이게 됩니다.

관리 부담은?

Supabase가 서버·백업 등을 알아서 해주는 관리형 서비스라, 우리가 따로 서버를 돌볼 필요가 없습니다. 무료 범위 안에서 쓰고 있어요.
